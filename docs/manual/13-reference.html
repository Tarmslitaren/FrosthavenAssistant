§13.3Icon and symbol glossary

The app's icons are sourced from PNG assets under frosthaven_assistant/assets/images/. The glossary below covers the icons that appear most often during play; specialized icons (boss frames, edition-specific variants) are described in their relevant sections.

GroupIconsWhere they appear
ElementsFire (flame), Ice (snowflake), Air (swirl), Earth (leaf), Light (radiating star), Dark (crescent moon)Top bar (§3.2), monster ability cards.
ConditionsWound (blood drop), Poison (drop with skull), Stun (lightning), Disarm (broken sword), Immobilize (chains), Muddle (X), Strengthen (up arrow), Bless (orb with rays), Curse (dark orb), Invisible (eye)Character banners, standee menus.
Scenario statsScenario level (number), Trap (triangle), Hazardous terrain, XP per kill (star + plus), Coin multiplier (coin + ×N)Bottom bar (§3.4).
Class glyphsPer-class icons (Brute axe, Spellweaver runes, Tinkerer gears, etc.); colored to match the class banner colorCharacter banners, condition tokens, character library.
Modifier deck symbols+0 / +1 / +2 / −1 / −2, Null, ×2, Rolling-modifier ribbon, Bless / CurseAMD cards.
Monster type indicatorsNormal frame (muted), Elite frame (yellow border), Boss frame (custom)Standee blocks on monster widgets.
Loot deckMoney, Lumber, Hide, Metal, Arrowvine, Axenut, Corpsecap, Flamefruit, Rockroot, SnowthistleLoot deck menu (§7.1); Frosthaven only.

Frosthaven uses slightly different art for some conditions (the codebase distinguishes via the _fh suffix on icon paths in condition_icon.dart). The active variant is picked automatically based on scenario edition.

§13.4Keyboard shortcuts (desktop)

Implemented in lib/Layout/global_hotkeys.dart. Shortcuts are suppressed when a text input is focused (the numpad's text field, the settings menu inputs, etc.).

KeyAction
SpaceDraw / Next Round (the morphing button) — §5.2
TabAdvance the active figure indicator — useful for sweeping through the initiative order.
Shift+TabReverse the active figure indicator (undo last activation).
Ctrl/⌘+ZUndo (§10.1).
Ctrl/⌘+YRedo.
1 – 6Toggle elements (1: fire, 2: ice, 3: air, 4: earth, 5: light, 6: dark). Same lifecycle as tapping in the top bar — §5.7.

Cross-reference upstream PR #304 for shortcut history and any pending additions.

§13.5Where to get help

  • GitHub issues: github.com/Tarmslitaren/FrosthavenAssistant/issues for bug reports and feature requests.
  • BoardGameGeek: the X-Haven Assistant thread on the Gloomhaven and Frosthaven BGG forums is where ad-hoc Q&A actually lives — search there before opening an issue if your question might already be answered.
  • Maintainer email: in the project README, for questions that don't fit a GitHub issue (sponsorship, app store concerns, etc.).

§13.6Known issues

The README's Known Issues section is the authoritative list — refer there rather than duplicating it here, since it changes per release. As of this writing, the most relevant known issue for this manual is #135 (the Draw / Next Round morphing-button race), called out throughout in the relevant sections.
