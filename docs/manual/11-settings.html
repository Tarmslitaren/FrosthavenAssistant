§11Settings Reference

Sidebar drawer → Settings opens the settings menu (SettingsMenu, lib/Layout/menus/settings_menu.dart). The README enumerates every option; this section covers the ones that materially change app behavior. The table below is grouped by intent.

SettingTypeWhat it doesDefault
§11.1Display and layout
Dark mode Toggle Switches the bar backgrounds and overall palette to a darker variant. Independent of the Frosthaven style (also in this menu). Off
App Bar Scaling Slider Resizes the top and bottom bars. Useful on tablets where the default is too small. The corresponding userScalingBars value listenable cascades to most widget sizes. 1.6 (desktop) / 1.0 (mobile)
Main List Scaling Slider Resizes the main list (banners + monster widgets). Set independently from bar scaling. 1.0
Fullscreen Toggle (desktop / Android) Hides the OS title bar; useful for game-day setups. Not shown on iOS. On
Stat card text shimmers Toggle Cosmetic — animates monster ability text. Some find it festive; others find it distracting. On (desktop) / Off (mobile)
§11.2Round-flow behavior
Soft numpad for input Toggle When on, tapping an initiative slot opens the in-app numpad (§5.3). When off — the default — tapping the slot opens your OS numeric keyboard instead. Off
Don't ask for initiative Toggle Skips the initiative-entry step entirely; useful for groups using a separate initiative tool or playing with always-open initiative. On
Expire Conditions Toggle Auto-decrements end-of-round conditions per the rulebook (§10.3). On
Auto Add Standees Toggle Triggers the auto-spawn dialog (§6.4) when sections introduce monsters. On
Auto Add Timed Spawns Toggle Same idea but for scenarios with time-based (round-N) spawns rather than section-based. On
Random Standees Toggle Randomizes which standee number is added rather than picking the lowest available. Tabletop-style draw-from-bag emulation. Off
Use Frosthaven Hazardous Terrain Calculation in OG Gloomhaven Toggle Applies the Frosthaven hazardous-terrain damage rule (per-step) to original Gloomhaven scenarios as well. Off restores the legacy Gloomhaven calculation. Field: fhHazTerrainCalcInOGGloom. On
§11.3Display filters and content
Don't track Standees Toggle Hides standee numbers and presents monster types as aggregate HP pools (§10.3). Off
No Calculations Toggle Disables the auto-applied modifier-card math; players resolve attack modifiers manually. Used by groups who prefer to do the math themselves. Off
Hide Loot Deck Toggle Hides the Frosthaven loot-deck UI even on Frosthaven scenarios. For groups who want to track loot by hand. Off
Show Scenario names in list Toggle Adds the scenario name to the title; useful when running multiple groups. On
Show Battle Goal Reminder Toggle Surfaces a battle-goal toast at scenario start. On
Show Attack Modifier Decks Toggle Toggles AMD visibility globally. On
Show Custom Content Toggle Enables fan-made content (Crimson Scales, Trail of Ashes, Seeker of Xorn, CCUG editions). On
Show Sections in Main Screen Toggle Shows scenario section markers (auto-spawn triggers, see §6.4) inline in the main list. Off hides the section row. Field: showSectionsInMainView. On
Show Round Special Rule Reminders Toggle Surfaces round-special-rule reminders during scenarios (§6.3). Field: showReminders. On
Show character Attack Modifier Decks Toggle Toggles per-character AMD visibility (independent of the global Show Attack Modifier Decks toggle above). Field: showCharacterAMD. On
Enable health wheel Toggle Shows the radial health-wheel control on character/monster widgets for tap-to-adjust HP. Off falls back to plus/minus controls only. Field: enableHeathWheel. On
Use Ally Attack Modifier Deck in OG Gloomhaven Toggle Applies the shared ally AMD (Frosthaven-style) to original Gloomhaven scenarios. Off keeps OG Gloomhaven without an ally deck. Field: allyDeckInOGGloom. On
Unlock specials Action Opens the special-unlocks dialog, which reveals the hidden Frosthaven character classes (those that show as ??? in §4.1). —
§11.4Networking
Network behavior — last-known address, startup connect, server lifecycle — is documented in §9. There is no auto-reconnect on foreground; Connect on startup below is the only automatic reconnect path. Controls below.
Connect as Client Toggle Connects this device to a host using the IP and port below. Disabled while a server is running locally; status row shows Connecting… / Connected. Off
server ip address Text field Last-known host address used by the client connect button. 192.168.1.???
port Text field (numeric) TCP port for client connect / host server. 4567
Start Host Server Toggle Starts a host on this device using the port above; flips to Stop Server while running. Off
Outgoing IPv6 Dropdown Picks which local interface to advertise when hosting (multi-homed machines only). First detected interface
Connect on startup Toggle If on, the app attempts a single client connect to the last-known address roughly two seconds after launch (connectClientOnStartup in lib/Resource/settings.dart). Does not run on resume from background — see §9.4. Off
§11.5State management
Load/Save State Action Opens the save menu. Save the entire current game state to a file (or load from one). Useful for moving a session between devices, sharing a problem state for bug reports, or resuming a paused scenario weeks later. —
